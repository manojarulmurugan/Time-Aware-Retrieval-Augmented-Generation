# Time-Aware RAG: Temporal Question Answering with Fine-tuned Contriever

Standard dense retrievers find semantically similar passages regardless of time period. This project fine-tunes `facebook/contriever-msmarco` with temporal triplet training and adds MRAG re-ranking (sliding-window MaxSim × temporal decay) to anchor retrieval to the correct historical era.

**Live demo:** [huggingface.co/spaces/manojarulmurugan/time-aware-rag](https://huggingface.co/spaces/manojarulmurugan/time-aware-rag)

---

## Results

Evaluated on ChroniclingAmericaQA (CAQA) validation split, 12,695 passages, year-explicit questions only.

| Configuration | Hit@1 | Hit@5 | Hit@10 | MRR@10 |
|---|---:|---:|---:|---:|
| Base Contriever (no MRAG) | 40.4% | 57.2% | 63.8% | 47.7% |
| Time-Aware Contriever (no MRAG) | 47.8% | 66.4% | 71.2% | 55.8% |
| Base + MRAG | 57.3% | 73.5% | 76.8% | 64.2% |
| **Time-Aware + MRAG** | **59.1%** | **75.3%** | **79.3%** | **65.7%** |

SQuAD temporal-filtered subset:

| Configuration | Hit@1 | Hit@5 |
|---|---:|---:|
| Base Contriever | 71.7% | 92.1% |
| Base + MRAG | 78.3% | 94.3% |
| Time-Aware + MRAG | **78.5%** | 94.2% |

## Architecture

```
FineWeb passages (20k)
        |
  T5 question generation  -->  (question, passage) pairs
        |
  Hard negative mining via base Contriever FAISS index
        |
  Triplet training (temporal + MS MARCO hybrid)
        |
  Fine-tuned Contriever  -->  contriever_finetuned_NEW_20k/
        |
  ┌─────────────────────────────────────────────┐
  │               Inference Pipeline            │
  │                                             │
  │  Query  -->  FAISS (top-20 candidates)      │
  │          -->  MRAG re-ranking               │
  │               - Sliding-window sentence     │
  │                 embeddings (pre-computed)   │
  │               - MaxSim × temporal decay     │
  │          -->  Groq LLM (grounded answer)    │
  └─────────────────────────────────────────────┘
```

**Training data:** 20k FineWeb-Edu passages filtered for year mentions (1800-2024). T5 generates temporal question-passage pairs. Hard negatives are semantically similar passages from the wrong time period, mined via base Contriever. 1,000 MS MARCO triplets are mixed in (50/50 hybrid) to prevent forgetting general retrieval ability.

**MRAG re-ranking:** Each passage is split into overlapping 3-sentence windows, each encoded into a 768-dim vector at startup. At query time, MaxSim finds the most relevant sentence cluster within each passage, multiplied by a triangular temporal decay score based on year distance from the query target year. Window embeddings are pre-computed once and stored as a tensor.

## Repository structure

```
src/
  fineweb_loader.py          # Download and filter FineWeb training corpus
  question_generation.py     # T5 temporal question generation
  contriever_training.py     # Triplet fine-tuning with hard negative mining
  chroniclingqa_eval.py      # CAQA evaluation harness (builds FAISS indices)
  mrag_integration.py        # MRAG: window embeddings, MaxSim, temporal decay
  squad_time_filter_eval.py  # SQuAD temporal subset evaluation
  pipeline.py                # Deployment: dual FAISS retrieval + MRAG at inference
  generator.py               # Groq LLM answer synthesis

api/
  main.py                    # FastAPI server (lifespan startup, /query, /health)

frontend/
  index.html                 # Single-file vanilla JS/CSS demo UI

scripts/
  01_setup_environment.sh
  02_generate_questions.sh
  03_train_contriever.sh
  04_evaluate_chroniclingqa.sh
  05_mrag_caqa_eval.sh
  06_mrag_squad_eval.sh
  colab_compute_window_embeddings.ipynb  # A100 Colab notebook for window state

demo_data/
  passages.json              # 12,695 CAQA validation passages
  base.index                 # FAISS index (base Contriever)
  timeaware.index            # FAISS index (fine-tuned model)

configs/
  config.yaml

outputs/
  mrag_eval_results_fast.json
  chroniclingqa_results.json
  mrag_caqa_results.json
```

## Running the research pipeline

```bash
pip install -r requirements.txt

# Step 1: prepare FineWeb data
./scripts/01_setup_environment.sh

# Step 2: generate temporal training questions
./scripts/02_generate_questions.sh

# Step 3: fine-tune Contriever (GPU required, ~2-4 hours)
./scripts/03_train_contriever.sh

# Step 4: evaluate on ChroniclingAmericaQA
./scripts/04_evaluate_chroniclingqa.sh

# Step 5: MRAG evaluation on CAQA
./scripts/05_mrag_caqa_eval.sh

# Step 6: SQuAD temporal evaluation
./scripts/06_mrag_squad_eval.sh
```

Or run everything with:
```bash
./scripts/run_complete_pipeline.sh
```

## Running the demo locally

The demo requires pre-computed FAISS indices and window embeddings in `demo_data/`. These are generated by the research pipeline or can be computed separately:

```bash
# 1. Compute window embeddings (run on GPU, ~5 min on A100)
#    Open scripts/colab_compute_window_embeddings.ipynb in Colab

# 2. Start the API server
uvicorn api.main:app --host 0.0.0.0 --port 7860 --workers 1

# 3. Open http://localhost:7860
```

For answer generation, set `GROQ_API_KEY` in your environment. Free API keys are available at console.groq.com. Retrieval works without a key.
